# 路由使用

## 路由组成

项目路由由三部分组成：

- 静态路由：`src/router/static-routes.ts`
- 模块路由：`src/router/modules/*.ts`
- 动态菜单路由：`src/router/dynamic-routes.ts`

## 1. 静态路由

`static-routes.ts` 定义登录页、403、404 和根布局路由：

- `Login`：公开路由（`meta.public = true`）
- `Root`：后台主布局，`children` 来自 `rootRouteModules`
- `Forbidden` / `NotFound`：异常页面

## 2. 模块路由（Root.children）

### 放置位置

- `src/router/modules`

### 自动聚合机制

- 聚合文件：`src/router/module-routes.ts`
- 使用 `import.meta.glob('./modules/**/*.ts', { eager: true })`
- 支持两种导出方式：
  - `export default RouteRecordRaw | RouteRecordRaw[]`
  - `export const routes: RouteRecordRaw | RouteRecordRaw[]`

### 新增模块示例

```ts
import type { RouteRecordRaw } from "vue-router";

export const routes: RouteRecordRaw[] = [
  {
    path: "demo",
    name: "DemoPage",
    component: () => import("@/views/dashboard/DashboardPage.vue"),
    meta: { title: "模块示例", requiresAuth: true },
  },
];
```

## 3. 动态菜单路由

- 来源：`authStore.fetchProfile()` 返回的 `menus`
- 转换：`menuStore.dynamicRoutes`（内部调用 `buildRoutesFromMenus`）
- 注入：`permissionStore.mountDynamicRoutes(router, routes)`

## 4. 三级路由

支持多级 `children` 递归：

- 菜单转路由：`dynamic-routes.ts` 递归
- 侧边栏渲染：`SidebarItem.vue` 递归
- 面包屑计算：`menu.ts` 递归查找链路

## 5. 退出登录路由清理

`permission store` 记录 `router.addRoute` 的移除函数。
退出登录时调用 `reset()`，会执行所有移除回调，真正删除动态路由。
