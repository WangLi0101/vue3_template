# 00-环境准备与首次启动

本文档适用于首次接手本项目的开发者，目标是帮助你在本地快速完成环境准备、编辑器配置和项目启动。

## 1. 环境要求

项目当前要求如下：

- `Node.js`: `^20.19.0 || >=22.12.0`
- `pnpm`: `>=10 <11`

仓库内已提供 [.nvmrc](../.nvmrc)，当前建议直接使用：

- `Node.js 22.17.1`

如果你使用 `nvm` 管理 Node 版本，可在项目根目录执行：

```bash
nvm use 22.17.1
```

## 2. 安装 pnpm

如果本机尚未安装 `pnpm`，可先执行：

```bash
npm install -g pnpm
```

安装完成后建议先确认版本：

```bash
node -v
pnpm -v
```

## 3. 用 VS Code 打开项目

请直接使用 VS Code 打开项目根目录：

```text
C:\Users\WangLi\Desktop\vue3_template
```

不要只打开 `src` 或 `docs` 子目录，否则工作区推荐插件、格式化配置和路径别名提示可能不会完整生效。

## 4. 安装推荐插件

仓库已经提供 VS Code 工作区推荐插件，配置文件位于：

- [extensions.json](../.vscode/extensions.json)
- [settings.json](../.vscode/settings.json)

推荐安装方式：

1. 打开项目后，若右下角出现“推荐扩展”提示，直接点击安装。
2. 如果没有弹窗，按 `Ctrl + Shift + P`。
3. 执行 `Extensions: Show Recommended Extensions`。
4. 安装当前工作区推荐的全部扩展。

当前推荐插件如下：

- `Vue.volar`
- `dbaeumer.vscode-eslint`
- `esbenp.prettier-vscode`
- `stylelint.vscode-stylelint`
- `bradlc.vscode-tailwindcss`
- `EditorConfig.EditorConfig`

注意事项：

- Vue 3 项目请使用 `Volar`，不要与 `Vetur` 同时启用。
- 安装完插件后，建议执行一次 “Developer: Reload Window” 让工作区配置完整生效。

## 5. 工作区默认行为

当前工作区已预置常用编辑器设置，安装推荐插件后通常无需再手动额外配置：

- 保存时自动格式化
- `ESLint` 修复可自动应用
- `Stylelint` 修复可自动应用
- `Prettier` 作为默认格式化器
- 行尾统一为 `LF`
- 自动补齐文件末尾换行并清理多余空格

相关配置来源：

- [settings.json](../.vscode/settings.json)
- [.editorconfig](../.editorconfig)

## 6. 安装依赖并启动项目

在项目根目录执行：

```bash
pnpm install
pnpm dev
```

启动成功后，根据终端输出的本地地址在浏览器中访问项目。

## 7. 常用开发命令

```bash
pnpm run typecheck
pnpm run lint
pnpm run lint:style
pnpm run check
pnpm run format
pnpm run format:check
```

命令说明：

- `pnpm run typecheck`：执行 TypeScript 类型检查
- `pnpm run lint`：执行 ESLint 检查
- `pnpm run lint:style`：执行样式检查
- `pnpm run check`：执行综合检查，当前等于 `typecheck + lint + lint:style`
- `pnpm run format`：格式化项目文件
- `pnpm run format:check`：检查文件是否符合格式规范

## 8. 首次启动后的建议验证

建议在第一次拉起项目后顺手验证以下内容：

- 页面是否可以正常打开
- 浏览器控制台是否存在报错
- 保存 `.vue` 文件时是否会自动格式化
- `ESLint`、`Stylelint` 是否能在编辑器中正常提示
- `pnpm run check` 是否可以通过

## 9. 常见问题

### 9.1 没有收到 VS Code 插件推荐弹窗

可手动打开命令面板并执行：

```text
Extensions: Show Recommended Extensions
```

### 9.2 保存文件时没有自动格式化

优先检查以下几点：

- 是否安装了 `esbenp.prettier-vscode`
- 是否安装了 `dbaeumer.vscode-eslint`
- 是否安装了 `stylelint.vscode-stylelint`
- 是否以项目根目录作为工作区打开
- 是否安装插件后尚未重载 VS Code 窗口

### 9.3 `pnpm` 命令无法使用

通常是因为本机未安装 `pnpm`，或安装后终端尚未刷新。可重新打开终端后再执行：

```bash
pnpm -v
```
