# 开发与扩展

## 目录约定

### 路由

- `src/router/static-routes.ts`：固定入口路由
- `src/router/module-routes.ts`：聚合模块路由
- `src/router/modules/*.ts`：静态模块路由定义
- `src/router/dynamic-routes.ts`：后端菜单转动态路由

### 状态

- `src/stores/modules/auth.ts`
- `src/stores/modules/permission.ts`
- `src/stores/modules/menu.ts`
- `src/stores/modules/tabs.ts`
- `src/stores/modules/ui.ts`
- `src/stores/modules/theme.ts`

### 接口

- `src/api/<domain>/api.ts`：接口函数
- `src/api/<domain>/types.ts`：领域类型
- `src/api/<domain>/index.ts`：统一导出
- `src/utils/http/*`：通用请求能力

### 视图与布局

- `src/views/*`：业务页面
- `src/layout/*`：布局壳与导航组件
- `src/components/AppRouterView.vue`：通用路由容器
- `src/composables/*`：与布局交互相关的组合式逻辑

### 权限

- `src/directives/modules/*.ts`
- `src/directives/index.ts`

## 新增一个后台页面

1. 在 `src/views/...` 下新增页面组件。
2. 选择接入方式：
   - 固定路由：在 `src/router/modules/*.ts` 增加模块路由
   - 动态菜单路由：由后端或 Mock 的 `menus` 返回对应菜单节点
3. 如需页面级权限，给路由增加 `meta.permission`。
4. 如需按钮级权限，在模板中增加 `v-permission` 或 `v-role`。
5. 如需出现在导航中，补齐 `meta.title`，按需补 `icon`、`rank`、`hidden`。

## 新增一个静态模块路由

在 `src/router/modules` 新建文件并导出 `routes` 或 `default`：

```ts
import type { RouteRecordRaw } from "vue-router";

export const routes: RouteRecordRaw[] = [
  {
    path: "report",
    name: "ReportPage",
    component: () => import("@/views/report/ReportPage.vue"),
    meta: {
      title: "报表中心",
      rank: 10,
      icon: "mdi:file-chart-outline",
    },
  },
];
```

建议：

- 需要多级嵌套时使用 `AppRouterView`
- 需要出现在侧边栏时必须提供 `meta.title`
- 排序使用 `meta.rank`

## 新增一个动态菜单页面

前端需要完成两件事：

1. 新建页面组件，例如 `src/views/report/ReportPage.vue`
2. 确保后端或 Mock 返回的菜单节点可以映射到该组件

菜单示例：

```ts
{
  id: "500",
  name: "ReportPage",
  path: "report",
  component: "report/ReportPage",
  meta: {
    title: "报表中心",
    permission: "report:view",
    rank: 10
  }
}
```

约束：

- `name` 需要稳定且唯一
- `path` 建议和导航语义一致
- `component` 以 `src/views` 为根路径映射，不带 `.vue`
- 容器菜单可使用 `component: "router-view"` 或省略 `component` 并保留 `children`

## 新增一个接口域

推荐结构：

```text
src/api/example/
  api.ts
  types.ts
  index.ts
```

建议步骤：

1. 在 `types.ts` 定义请求和响应类型
2. 在 `api.ts` 中通过 `request<T>()` 编写接口函数
3. 如需新服务地址，在 `src/utils/http/config.ts` 中增加 `ServiceName`
4. 若需本地联调，在 `mock/` 中同步补接口

## 新增一个权限点

页面级权限：

- 给路由加 `meta.permission`
- 由守卫使用 `permissionStore.hasAll()` 校验

按钮级权限：

- 使用 `v-permission="'sys:user:create'"`
- 或使用 `v-role="['super_admin']"`

同步要求：

- 后端或 Mock 的 `permissions` 必须返回该权限码
- 如果页面需要在菜单中展示，也要同步补 `menus`

## 主题与界面能力扩展

### 扩展主题色

- 修改 `src/config/theme.ts` 中的 `themeColorPresets`
- 如需新增变量，同时补 `buildPrimaryCssVars()` 或 `buildSemanticCssVars()`

### 扩展持久化 UI 状态

- 在 `src/stores/modules/ui.ts` 中扩展 `PersistedUiState`
- 保持读取和写入逻辑对齐，避免旧缓存解析失败

### 扩展标签页行为

- 标签页数据源位于 `src/stores/modules/tabs.ts`
- 自动追踪逻辑位于 `src/composables/useRouteTabTracking.ts`

## 开发检查建议

- 类型检查：`pnpm run typecheck`
- Lint 检查：`pnpm run lint`
- 综合检查：`pnpm run check`
- 格式检查：`pnpm run format:check`

关键链路建议每次都手工验证一次：

- 登录
- 首次受保护页面初始化
- 动态路由注入
- 根路径跳首页
- 标签页增删
- 权限按钮显示
- 退出登录后的路由清理
