# 06.代码规范

## 1. 目标与范围

本项目使用 `ESLint + Prettier + TypeScript + EditorConfig` 作为统一规范方案，目标是：

- 减少多人协作时的风格漂移
- 尽早发现语法、类型和常见逻辑问题
- 让提交和评审更聚焦于真实业务改动

适用范围：

- `src/`
- `mock/`
- `docs/`
- 根目录配置文件中的 `js/ts/vue/md/json/cjs` 等文件

## 2. 执行命令

```bash
pnpm run typecheck # 执行 TypeScript 类型检查
pnpm run lint # 执行 ESLint 检查
pnpm run lint:fix # 自动修复可处理的 ESLint 问题
pnpm run format # 使用 Prettier 格式化项目文件
pnpm run format:check # 检查项目文件是否符合 Prettier 规范
pnpm run check # 执行综合检查（typecheck + lint）
pnpm run prepare # 安装或刷新 Husky Git hooks
```

说明：

- `lint` 使用 `--max-warnings=0`
- `check` 当前等于 `typecheck + lint`
- `format:check` 没有并入 `check`，需要单独执行

## 3. Git Hooks（Husky）

项目已接入 Husky。

当前相关配置：

- `package.json`：`prepare: "husky"`
- Hook 文件：`.husky/pre-commit`

`pre-commit` 当前执行：

```bash
pnpm run check # 提交前执行类型和 Lint 综合检查
pnpm run format:check # 提交前检查格式是否符合 Prettier 规范
```

执行策略：

- 任一步骤失败都会阻止提交
- 不建议跳过 Hook；如确需跳过，应在提交说明里明确原因

## 4. ESLint 规范

配置文件：`eslint.config.mjs`

### 4.1 基础配置

- 使用 ESLint Flat Config
- 组合规则：
  - `@eslint/js`
  - `typescript-eslint`
  - `eslint-plugin-vue`
  - `eslint-config-prettier`
- 语言环境：
  - `ecmaVersion: "latest"`
  - 默认 `sourceType: "module"`
  - `*.cjs` 例外为 `commonjs`
  - 全局变量：`browser + node`

### 4.2 忽略路径

- `dist/**`
- `node_modules/**`
- `.pnpm-store/**`
- `coverage/**`
- `*.min.js`

### 4.3 关键规则

错误级别：

- `curly: ["error", "all"]`
- `eqeqeq: ["error", "always"]`
- `no-debugger: "error"`
- `@typescript-eslint/consistent-type-imports: "error"`
- `@typescript-eslint/no-unused-vars: "error"`
- `vue/block-order: "error"`
- `vue/padding-line-between-blocks: ["error", "always"]`
- `vue/multi-word-component-names: "error"`

补充说明：

- `_` 前缀的未使用参数、变量、异常参数允许保留
- `vue/multi-word-component-names` 当前仅忽略 `Index`

警告级别：

- `no-console: "warn"`，只允许 `console.warn`、`console.error`
- `vue/no-v-html: "warn"`

### 4.4 项目级例外

为兼容当前图标实现，以下文件做了定点例外：

- 文件：`src/components/ReIcon/src/hooks.ts`
- 规则：`vue/one-component-per-file = off`

除该文件外，不建议继续扩大豁免范围。

## 5. Prettier 规范

配置文件：`prettier.config.mjs`

当前主要参数：

- `printWidth: 100`
- `tabWidth: 2`
- `useTabs: false`
- `semi: true`
- `singleQuote: false`
- `trailingComma: "all"`
- `arrowParens: "always"`
- `endOfLine: "lf"`
- `vueIndentScriptAndStyle: false`

插件：

- `prettier-plugin-tailwindcss`

说明：

- Tailwind 类名会在格式化时自动排序
- 与 ESLint 的风格冲突由 `eslint-config-prettier` 兜底

## 6. EditorConfig 规范

配置文件：`.editorconfig`

统一基础行为：

- `charset = utf-8`
- `end_of_line = lf`
- `indent_style = space`
- `indent_size = 2`
- `insert_final_newline = true`
- `trim_trailing_whitespace = true`

特殊规则：

- Markdown 文件保留行尾空格：`[*.md] trim_trailing_whitespace = false`

## 7. 团队协作建议

建议本地开发流程：

1. 开发完成后执行 `pnpm run format`
2. 再执行 `pnpm run check`
3. 提交前执行 `pnpm run format:check`

评审时重点关注：

- 是否通过类型和 Lint 校验
- 是否新增了不必要的规则例外
- 是否出现 `any`、无意义断言或深层路径硬编码
- 文档、Mock、类型和实现是否同步更新
