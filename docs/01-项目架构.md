# 项目架构

## 项目定位

该项目是一个基于 `Vue 3 + Vite + TypeScript + Pinia + Element Plus` 的 RBAC 管理后台模板，目标是提供一套可直接扩展的后台基础骨架，而不是只演示单一页面。

当前重点能力：

- 登录态管理与本地持久化
- 基于角色、权限码、菜单树的访问控制
- 静态模块路由与后端菜单动态路由并存
- 侧边栏、面包屑、标签页统一联动
- 主题模式和主题色动态切换
- 基于 `vite-plugin-mock` 的本地联调能力

## 启动流程

应用入口位于 `src/main.ts`，当前初始化顺序如下：

1. `createApp(App)` 创建应用实例
2. `setupRouter(app)` 注册路由
3. `setupStore(app)` 注册 Pinia
4. `useThemeStore(store).initTheme()` 初始化主题变量和系统主题监听
5. `setupRouterGuards(router)` 注册全局路由守卫
6. `app.use(ElementPlus, { locale: zhCn })`
7. `setupDirectives(app)` 注册 `v-permission`、`v-role`
8. `app.mount("#app")`

这个顺序的意义：

- 路由和 store 先注册，守卫和指令才能安全访问 store
- 首屏挂载前先应用主题，减少模式切换闪烁
- 路由守卫在应用渲染前接管登录态和动态路由初始化

## 分层结构

### 1. 启动与应用壳层

目录与文件：

- `src/main.ts`
- `src/App.vue`
- `src/layout/AppLayout.vue`

职责：

- 组装应用运行时依赖
- 提供后台基础布局壳层
- 承接头部、侧边栏、内容区和路由视图

### 2. 路由层

目录：`src/router`

核心文件：

- `index.ts`：创建 `router`
- `static-routes.ts`：定义登录页、403、根布局、404
- `module-routes.ts`：自动聚合 `src/router/modules/**/*.ts`
- `dynamic-routes.ts`：将后端 `menus` 转为 `RouteRecordRaw[]`
- `guard.ts`：处理登录校验、会话初始化、动态路由挂载和权限校验
- `modules/nested.ts`：静态模块路由示例

当前路由模型：

- 静态路由在应用启动时就已注册
- 模块路由作为 `Root.children` 的静态子路由参与菜单渲染
- 动态菜单路由在首次进入受保护页面时挂载到 `Root`

### 3. 状态管理层

目录：`src/stores/modules`

当前 store 划分：

- `auth.ts`
  - 管理当前用户、会话初始化状态
  - 提供 `login()`、`initializeSession()`、`logoutLocal()`
- `permission.ts`
  - 管理 `roles`、`permissions`
  - 提供 `hasAny()`、`hasAll()`、`hasRole()`
  - 负责动态路由挂载和卸载
- `menu.ts`
  - 保存后端原始菜单 `rawMenus`
  - 计算 `sidebarMenus`、`dynamicRoutes`
  - 提供 `getBreadcrumbs()`、`resolveRootRedirectTarget()`
- `tabs.ts`
  - 管理头部标签页
  - 自动同步首页标签
  - 过滤公开页、隐藏页、403、404
- `ui.ts`
  - 维护侧边栏折叠状态
  - 使用 `localStorage` 做轻量持久化
- `theme.ts`
  - 管理主题模式、主题主色、系统主题联动
  - 负责向根节点写入 CSS 变量和 Element Plus 主题变量

整体原则：

- `auth` 只管登录态和会话初始化
- `permission` 只管权限集合和动态路由生命周期
- `menu` 只管菜单衍生结果和导航体验
- `tabs/ui/theme` 管界面状态，不混入业务鉴权逻辑

### 4. 接口与 HTTP 层

目录：

- `src/api`
- `src/utils/http`
- `src/types/http.ts`

当前分层：

- `src/api/auth/api.ts`
  - 只定义认证域接口函数
- `src/api/auth/types.ts`
  - 定义登录、用户、角色、权限、菜单相关类型
- `src/utils/http/client.ts`
  - 创建 Axios 实例，统一超时设置
- `src/utils/http/interceptors.ts`
  - 自动挂载 `Authorization`
  - 统一处理业务错误和 HTTP 错误
- `src/utils/http/request.ts`
  - 通过 `ServiceName` 选择 `baseURL`
  - 统一返回 `ApiResponse<T>` 或下载响应
- `src/utils/http/config.ts`
  - 定义服务名到基础地址的映射

当前接口约定：

- 统一响应结构为 `{ code, data, message }`
- 认证相关接口拆成 5 个独立接口：
  - `POST /api/auth/login`
  - `GET /api/auth/me`
  - `GET /api/auth/roles`
  - `GET /api/auth/permissions`
  - `GET /api/auth/menus`

### 5. 权限与指令层

目录：`src/directives`

当前自定义指令：

- `v-permission`
  - 基于权限码控制元素显示
  - 使用 `permissionStore.hasAny()`
- `v-role`
  - 基于角色控制元素显示
  - 使用 `permissionStore.hasRole()`

实现特点：

- 在 `mounted` 阶段直接移除无权限元素
- 适合按钮级、局部区域级的体验控制
- 页面级权限仍然交给路由守卫处理

### 6. 布局与导航层

目录：

- `src/layout`
- `src/components`
- `src/composables`

核心组件：

- `src/layout/AppLayout.vue`
  - 负责整体左右布局和内容区骨架
- `src/layout/components/AppSidebar.vue`
  - 基于 `menuStore.sidebarMenus` 渲染侧边栏
- `src/layout/components/AppHeader.vue`
  - 承接标题、用户信息、系统配置抽屉和标签栏
- `src/layout/components/AppHeaderTags.vue`
  - 渲染可关闭标签页
- `src/layout/components/AppBreadcrumb.vue`
  - 根据当前路由和菜单结构生成面包屑
- `src/layout/components/AppSettingsDrawer.vue`
  - 切换主题模式和主题色
- `src/components/AppRouterView.vue`
  - 统一处理嵌套路由、页面过渡和 `KeepAlive`
- `src/composables/useRouteTabTracking.ts`
  - 监听当前路由并自动同步标签页

### 7. Mock 数据层

目录：`mock`

文件：

- `mock/auth.ts`
  - 提供认证域全部 mock 接口
- `mock/data/rbac-data.ts`
  - 提供账号、角色、权限和菜单演示数据

当前内置账号：

- `admin / admin123`
- `auditor / auditor123`

## 核心数据流

当前主链路如下：

1. 登录页调用 `authStore.login()` 获取 `accessToken + refreshToken` 并写入本地缓存
2. 用户进入非公开路由时，路由守卫检查 `accessToken`
3. 首次进入受保护页面时，`authStore.initializeSession()` 并行请求当前用户、角色、权限和菜单
4. `permissionStore` 写入角色和权限集合
5. `menuStore` 写入后端菜单，并基于菜单计算动态路由和侧边栏树
6. `tabsStore.syncHomeTag()` 根据首个可见菜单同步首页标签
7. `permissionStore.mountDynamicRoutes()` 将动态菜单路由挂载到 `Root`
8. 布局组件基于 `sidebarMenus`、`tabs`、`getBreadcrumbs()` 渲染导航
9. 若 `accessToken` 过期，请求拦截器会尝试使用 `refreshToken` 自动刷新后重放当前请求
10. 页面元素通过 `v-permission` 和 `v-role` 控制局部可见性

## 当前目录关注点

如果要扩展项目，通常按以下边界落点：

- 新增固定页面：`src/router/modules` + `src/views`
- 新增后端控制页面：后端补 `menus`，前端补对应 `src/views`
- 新增按钮权限：页面模板 + 权限码配置 + 后端接口鉴权
- 新增主题能力：`src/config/theme.ts` + `theme store`
- 新增接口域：`src/api/<domain>` + `src/utils/http`

## 关键设计原则

- 单一职责：认证、权限、菜单、标签页、主题分别由独立 store 维护
- 契约优先：前端消费明确的 `roles / permissions / menus` 契约，不在页面中临时拼装权限模型
- 菜单驱动导航：侧边栏、面包屑和首页入口都围绕统一菜单衍生
- 体验与安全分层：前端权限只负责导航和显示控制，最终安全边界仍在后端
