# 项目架构

## 目标

该项目是一个 Vue3 RBAC 管理后台模板，目标是提供以下能力：

- 登录鉴权（单 Token）
- 菜单级与按钮级权限控制
- 动态路由注入与退出清理
- 可扩展的模块路由与 Mock 接口

## 分层结构

### 1. 应用启动层

- 入口文件：`src/main.ts`
- 职责：初始化 Vue 应用、注册 Router/Pinia、注册 Element Plus 与自定义指令。

### 2. 路由层

- 目录：`src/router`
- 核心文件：
  - `index.ts`：创建并导出 router
  - `static-routes.ts`：静态路由（登录、403、Root、404）
  - `dynamic-routes.ts`：将后端菜单树转为 `RouteRecordRaw`
  - `module-routes.ts`：自动聚合 `router/modules` 下的静态模块路由
  - `guard.ts`：全局鉴权守卫与初始化逻辑

### 3. 状态管理层

- 目录：`src/stores/modules`
- 模块：
  - `auth.ts`：登录态、用户、角色、权限码、菜单原始数据
  - `menu.ts`：侧边栏菜单、面包屑、动态路由衍生数据
  - `permission.ts`：动态路由挂载与清理（基于 addRoute 返回回调）

### 4. 网络与接口层

- 目录：`src/api`
- 模块：
  - `request.ts`：Axios 封装，统一解包 `ApiResponse`
  - `auth.ts`：认证相关接口定义

### 5. 视图与布局层

- 目录：`src/layout`、`src/views`
- 能力：
  - `AppLayout` 提供后台主框架
  - `AppSidebar` 与 `AppHeader` 提供菜单和面包屑
  - `views/system/*` 作为系统管理页面示例

### 6. Mock 层

- 目录：`mock`
- 文件：
  - `auth.ts`：mock 登录与用户信息接口
  - `data/rbac-data.ts`：账号、角色、权限、菜单数据

## 关键设计原则

- 单一职责：菜单计算在 `menu store`，路由移除在 `permission store`
- 可扩展：`router/modules/*.ts` 自动聚合
- 可替换：Mock 与真实后端接口结构统一（`code/data/message`）
